# 分层混合架构  

> 本方案是我们的第三套方案设想，还未验证  

分层混合架构方案不使用纯端到端RL，也不使用纯传统算法  
使用分层混合架构的优点在于可以很好的区分每部分的职责与任务，同时也可以较为方便的进行模块化训练与优化  

## 架构总览  

> Layer 4：感知（特征提取、建图维护）  
> Layer 3: 决策（判断动作模式：探索、清扫、充电、避让）  
> Layer 2: 规划（根据决策生成局部目标点或路径）  
> Layer 1: 执行（进行动态避障和安全检查，生成最终动作）  

## Layer说明  

### Layer 4：感知  

> 栅格地图 + 历史记忆  

**具体任务：**  

1. 接收局部视野信息，识别道路、污渍、障碍物、官方机器人、充电桩  
2. 维护全局/局部栅格地图状态  
3. 记录已清扫区域与未探索区域  
4. 维护危险区域热度信息  
5. 输出统一状态特征供下层使用  

**暂定算法：**  

1. 建图与维护：  
   - 占据栅格地图（Occupancy Grid Map）  
   - 污渍状态图  
   - 风险热力图  
2. 历史记忆：LSTM / GRU  
3. 特征提取与融合：  
   - CNN  
   - Transformer  

### Layer 3：决策  

> PPO + LSTM  

**具体任务：**  

1. 判断当前行为模式  
2. 在探索、清扫、充电、避让之间切换  
3. 根据电量和环境状态进行高层任务分配  
4. 输出当前目标模式和目标区域  

**行为模式：**  

1. 探索模式  
2. 清扫模式  
3. 返航充电模式  
4. 动态避让模式  

**暂定算法：**  

1. PPO + LSTM  
2. 分层PPO  
3. PPO + Moe  

### Layer 2：规划  

> 路径规划 + 局部目标生成  

**具体任务：**  

1. 根据决策层模式生成局部目标点  
2. 规划从当前位置到目标点的最优路径  
3. 动态更新路径以避开移动障碍  
4. 优化清扫覆盖效率  

**暂定算法：**  

1. 全局路径规划：  
   - A*  
   - Dijkstra  
2. 局部动态规划：  
   - 局部 A*  
   - 滚动窗口重规划  
3. 覆盖式清扫规划：  
   - Coverage Path Planning  

### Layer 1：执行  

> CAS 动态避障 + 安全控制  

**具体任务：**  

1. 接收规划路径  
2. 进行动态障碍检测  
3. 安全检查与碰撞风险预测  
4. 输出最终控制动作  

**暂定算法：**  

1. 动态避障与路径追踪：  
   - SAC  
   - 局部安全约束控制  
2. 动作修正：  
   - 安全偏置过滤器  
   - 紧急制动机制  

## 训练与工程方案  

1. 对Layer 1进行单独训练，使其有跟踪 + 避障的能力，参考Layer 2的局部目标点/参考路径进行运动  
2. 【可选】对Layer 4进行预训练（自监督预训练等）  
3. Layer 4 + Layer 2 进行baseline测试，是否能够完成建图，完成正确路径规划（默认Layer 3和Layer 1均完全正确执行任务）  
4. 整合训练Layer 3决策
